# 风眼 · Typhoon Eye —— 项目计划书

> 台风季，信息要像灯塔一样：一眼可见、一句能懂、一步能做。
> 本项目提供台风实况信息的极简呈现，并按预警等级给出可执行的应急预案，帮助沿海居民在风暴来临前做对每一步。

- 仓库：https://github.com/Mr-Salticidae/typhoon-eye （public）
- 状态：v0.1 demo
- 立项日期：2026-07-10

---

## 一、背景与目标

每年 7–9 月是中国东南沿海台风登陆高峰。权威数据并不缺（中央气象台、各省水利厅都有发布），缺的是**普通人打开就能看懂、看完就知道该做什么**的页面：

- 官方页面信息密度高，老人与着急的人抓不到重点；
- "该不该封窗、该不该囤水、什么时候必须撤离"没有和预警等级直接挂钩；
- 移动端体验参差。

**目标**：做一个零依赖、秒开、离线可用的静态单页——

1. **看得懂**：台风叫什么、在哪、多强、往哪走、什么时候到，五个问题一屏答完；
2. **做得到**：按蓝→黄→橙→红四级预警给出递进式行动清单，勾选即存；
3. **信得过**：页面显著标注数据时效与来源，并始终引导用户以官方预警为最终依据。

## 二、用户与场景

| 用户 | 场景 | 关键诉求 |
|---|---|---|
| 沿海城市居民 | 台风预警发布后打开查看 | 强度、到达时间、我该做什么 |
| 在外子女 | 替老家父母查看并转述 | 一句话结论、可转发 |
| 社区工作者 | 组织防台准备 | 分级清单可当检查表用 |

## 三、功能范围

### v0.1 demo（本期交付）

- **实况面板**：台风名称/编号、中心位置、最大风力与风速、中心气压、移向移速、风圈半径；
- **路径可视化**：SVG 简化海岸线图，历史路径实线 + 预报路径虚线，当前位置脉动标记，逐点悬停查看时间与强度；
- **预警等级**：蓝/黄/橙/红四级切换，颜色与官方预警信号体系一致；
- **分级预案**：每级一份递进式行动清单（含上一级全部事项），可勾选，进度存 localStorage；
- **应急信息**：报警/急救/水上搜救电话、权威信息源直达链接；
- **双主题**：昼/夜皮肤，延续「蛛网之上」的纸墨设计体系；
- **演示数据**：v0.1 使用内置示例台风数据，页面显著标注"演示"。

### v0.2+（后续迭代，不在本期）

- 真实数据接入：GitHub Actions 定时抓取中央气象台/浙江台风路径系统公开数据，生成静态 JSON（绕开浏览器 CORS，且对源站零压力）；
- 多台风并存切换；城市定位与"距我多远"；预案清单打印版；PWA 离线缓存。

## 四、数据源策略

| 阶段 | 来源 | 方式 |
|---|---|---|
| v0.1 | 内置演示数据 `assets/data.js` | 结构对齐真实 API 字段，换源零改动 |
| v0.2 | 中央气象台 nmc.cn / 浙江省台风路径实时发布系统 | Actions 定时任务抓取 → 仓库内静态 JSON → Pages 分发 |

**红线**：本项目是信息聚合与科普工具，不是预警发布渠道。页面永远标注数据时间戳，并置顶提示"以当地政府与气象部门发布为准"。

## 五、设计语言

与「蛛网之上」同源：暖白纸面 / 墨色文字 / 单一强调色 + 细线分隔的杂志式排版；衬线标题、无衬线正文；`data-theme` 双皮肤。

差异点：强调色由蛛丝紫改为**风暴青**（海洋气象的色彩记忆），预警四色严格使用官方信号色，不做风格化，保证语义可靠。

极简、大气、内敛，但在预案区允许"热情"——用鼓励性的文案陪伴用户完成准备（"做完这几件事，你已经比 90% 的人更安全"）。

## 六、技术方案

- 纯静态：`index.html + assets/{style.css, app.js, data.js}`，零构建、零依赖、`file://` 双击可开；
- 路径图为手绘简化 SVG（非精确地图），只表达空间趋势，图上注明"示意"；
- 部署：GitHub Pages（main 分支根目录）；
- 兼容：现代浏览器 + 移动端优先的响应式；`prefers-reduced-motion` 降级动画。

## 八、风险与边界

1. **数据准确性**：演示数据 ≠ 实况。已用页头横幅 + 页脚双重免责声明。
2. **过度依赖风险**：任何第三方页面都可能滞后，预案区首条永远是"服从当地政府统一安排"。
3. **地图精度**：示意图不用于导航或撤离路线判断。
